# 明道云平台节点能力参考

> 此文件为平台能力知识库的索引。完整节点指南见 `references/manual.md` 和 workflow-nodes-complete-guide。

## 核心约束（来自元公理 B）

Agent 对明道云平台能力的认知仅限于已加载的文档。逻辑上合理的节点组合可能在平台不支持。

## 平台校验触发条件

设计涉及以下任何节点时，必须启动 Agent 2 (platform-verify.md)：
- 获取单条/多条数据节点
- 关联记录操作节点
- 子流程节点
- 审批节点

## 已知平台限制

> 此节从几建经验和 manual.md 中持续积累。

| 限制 | 说明 | 变通方案 |
|------|------|---------|
| 子流程并行执行 | 平台子流程默认逐条，不可并行 | 接受逐条，控制子流程数量 |
| 审批退回范围 | 仅支持退回发起节点 | 设计时确保审批链简单 |
| 字段显示上限 | 关联表显示字段有上限 | 只显示工作流需要的字段 |
| 批量操作上限 | 部分节点有批量处理上限 | 设计时考虑数据量 |

## 节点配置检查清单

Agent 2 (platform-verify.md) 逐节点检查：
1. 子模式是否存在（节点类型在平台支持的操作列表内）
2. 相邻节点数据链路是否合法（A节点输出→B节点输入的类型兼容）
3. 批量上限是否触发（数据量 > 平台限制时需分批处理）
4. 无数据处理策略是否配置（空数据→继续/中止）
