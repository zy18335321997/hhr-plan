# 供应链制造 APaaS 设计模式

> 来源：尚策项目反向蒸馏 — 136表 / 490工作流 / 8业务模块
> 适用范围：供应链协同、MRP物料需求、委外加工、采购-生产-仓储联动

---

## 一、核心特征：状态流转型

与莱尔德（数据加工型）不同，尚策是**状态流转型**系统：

| 特征 | 尚策 | 莱尔德 |
|------|------|--------|
| 创建 vs 更新 | create=132, update=690 | create=706, update=194 |
| 设计哲学 | 大量更新已有记录的状态/库存/进度 | 大量创建新记录并做数据运算 |
| 审批驱动 | 42条审批流(9%) | 10条审批流(2%) |
| 子流程复用 | 121条(25%) | 121条(28%) |

## 二、标准业务域模板

尚策比莱尔德多了两个域：**委外(外协加工)** 和 **问题反馈渠道**。

### 莱尔德六域 + 尚策扩展

```
# 莱尔德已有六域（同前）
销售域 / 采购域 / 生产域 / 仓储域 / 技术域 / 设备域

# 尚策新增域
委外域: 委外订单 → 委外发料 → 委外收货 → 委外质检 → 委外结算
人事域: 员工档案 → 考勤 → 请假 → 薪资
问题反馈: 问题上报 → 问题分类 → 处理派单 → 整改验收
```

## 三、供应链独有模式

### 3.1 MRP物料需求模式
```
物料需求清单 (18条工作流) → 采购物料明细 (13条) → 采购单 (11条)
```
- MRP计算逻辑分拆为多个子流程（MRP计算逻辑1/2/3）
- 物料短缺数运算独立为单独工作流
- 从需求→采购→收货，完整闭环

### 3.2 委外加工模式
```
委外订单 → 委外发料(物料出库) → 委外收货(成品入库) → 委外质检 → 委外结算
```
- 委外需要追踪物料发出和成品返回
- 结算按加工费+物料损耗计算
- 与内部生产共享工单模式但独立管理

### 3.3 询价-比价-定价模式
```
询价单(20条工作流) → 比价分析 → 供应商定价 → 采购价目清单
```
- 询价单工作流密度高(20条)，说明供应链中采购寻源是核心环节

## 四、尚策的教训（反面参考）

### 命名债务
- **49条工作流含"未命名"**（10%），包括"未命名子流程"33条、"未命名审批"10条
- 这是快速建设期留下的技术债，不是设计选择
- **教训**：每个子流程/审批块创建后立即命名，不要留到"以后"

### 模块划分过细
- 136张表中部分模块边界模糊（"其他"15张）
- 问题反馈渠道独立成域，但与质检功能重叠
- **教训**：模块不超过8个，超过则说明有合并空间

## 五、设计原则速查

| # | 原则 | 来源 | 做法 |
|---|------|------|------|
| 1 | 状态驱动 | update:create=5:1 | 工作流默认"修改已有记录"而非"创建新记录" |
| 2 | 审批前置 | 42条审批流 | 关键状态变更必须经过审批块 |
| 3 | 委外独立 | 11张委外表 | 外协加工独立成域，不与内部生产混用 |
| 4 | MRP分层 | 3个计算子流程 | 复杂计算逻辑分拆，单流控制在15节点内 |
| 5 | 询价闭环 | 询价→比价→定价→价目清单 | 采购寻源全链路覆盖 |
| 6 | 命名零容忍 | 49条未命名 | 创建后立即命名，不留技术债 |
