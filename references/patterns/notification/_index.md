# 通知策略模式目录

> 从几建 512 个 CRUD 字段映射中蒸馏。覆盖通知对象、时机、模板、渠道选择。

## 自动选择表

| 场景 | 模式 | 通知对象 | 典型时机 |
|------|------|---------|---------|
| 审批结果告知 | `approval-result` | 发起人 | 审批通过/否决后立即发送 |
| 任务分配告知 | `task-assignment` | 被分配人 | 工作流创建任务后发送 |
| 状态变更同步 | `status-sync` | 上下游相关人员 | 关键状态变更时 |
| 超时/异常告警 | `alert-escalation` | 管理者/拥有者 | 超时未处理或数据异常 |
| 周期性汇总 | `periodic-digest` | 按角色/部门 | 每日/每周定时汇总 |
| 多级通知链 | `cascade-notify` | 逐级通知 | 上级审批后通知下级执行 |

## 通知配置速查

| 模式 | 通知方式 | 模板要点 | 公理3要求 |
|------|---------|---------|---------|
| approval-result | 系统通知+邮件 | 【审批结果】+【字段名】内联引用 | 字段引用=业务上下文可读 |
| task-assignment | 系统通知 | 【任务名】+【截止日期】+【关联记录】 | 消息含可点击的工作流链接 |
| status-sync | 系统通知 | 【状态变更】+【操作人】+【时间】 | 状态值用中文，不用内部编码 |
| alert-escalation | 邮件+系统通知 | 【超时类型】+【已等待时长】+【处理入口】 | 含直接操作链接 |
| periodic-digest | 邮件 | 汇总表格+趋势+待处理项 | 模板=自定义Word模板 |
| cascade-notify | 系统通知 | 逐级通知，每级完成后触发下一级 | 每级明确标注来源 |

## 公理约束

| 约束 | 规则 |
|------|------|
| 公理3 | 通知消息必须用【字段名】内联引用流程字段值，不允许默认消息 |
| 公理5 | 通知发送失败不影响工作流继续执行 |
| 公理2 | 通知=知会，不=决策——通知对象不参与审批 |
