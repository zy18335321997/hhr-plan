# Approval-Result: 审批结果通知

> 审批通过/否决后立即通知发起人。最基本的通知模式。
> 节点签名: 审批→通知 | 节点数: 1-2

## 场景

- 审批通过→通知发起人"你的XX申请已通过"
- 审批否决→通知发起人"你的XX申请被否决，原因：..."
- 多级审批→每级审批后通知对应的下一级审批人

## 通知模板

```
通过模板:
【审批结果】你的【申请类型】已通过
审批人: 【审批人姓名】
审批时间: 【审批完成时间】
下一步: 【后续操作提示】

否决模板:
【审批结果】你的【申请类型】已被退回
审批人: 【审批人姓名】
退回原因: 【审批意见】
修改建议: 【如何修改后重新提交】
重新提交入口: [链接]
```

## 通知配置

| 配置项 | 值 |
|--------|-----|
| 通知对象 | 发起人（流程触发人） |
| 通知方式 | 系统通知（必选）+ 邮件（可选） |
| 通知时机 | 审批节点完成后立即发送 |
| 模板 | 自定义（非默认模板），含【字段名】引用 |

## 多级审批的通知识别

```
L1审批通过 → 通知L2审批人（你有新的审批待办）
L1审批否决 → 通知发起人（结束）
L2审批通过 → 通知发起人 + L3审批人（如存在）
最后一级通过 → 通知发起人 + 执行人
```

## 公理约束

| 约束 | 规则 |
|------|------|
| 模板 | 必须自定义，含【字段名】内联引用（公理3） |
| 通知失败 | 不影响工作流继续执行（公理5） |
| 通知对象 | 明确指定（发起人/审批人/指定角色），不群发 |
