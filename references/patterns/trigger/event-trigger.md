# Event-Trigger: 事件触发

> 数据变更自动触发级联操作。纯数据搬运，不涉及决策。
> 几建占比: 20%（66个事件触发工作流）

## 场景

- 新增记录→自动生成关联记录（入库→生成库存流水）
- 修改字段→自动同步状态（项目状态变更→更新关联任务状态）
- 删除记录→清理关联数据
- 审批通过→自动创建下游单据

## 配置模板

```
T0: 工作表事件触发
    工作表: [监听的表]
    事件类型: 新增记录 / 编辑记录 / 删除记录 / 审批通过
    监听字段: [具体字段]（编辑记录时必填，否则每次编辑都触发）
```

## 并发风险检查

| 场景 | 风险 | 缓解 |
|------|------|------|
| 同一表多个事件触发 | 同一操作可能同时激活多个工作流 | 标注 concurrent_trigger_risk |
| 多个事件流写同一字段 | 数据竞态 | 标注 write_conflict_risk，确认写入顺序 |
| 事件触发→触发另一事件→循环 | 触发风暴 | 检查触发链长度（公理4） |

## 事件链设计规则

```
1. 列出触发表的所有事件触发工作流
2. 检查每个事件流写入的字段是否有交集 → 标注冲突
3. 确认事件链不会形成循环（A事件→B事件→A事件）
4. 事件链深度 >3 → 考虑合并或加延时
```

## 公理约束

| 约束 | 规则 |
|------|------|
| 不允许替代人工判断 | 审批类→必须按钮，不用事件（公理2） |
| 空数据处理 | 查询/获取空→继续执行（公理5/定理4） |
| 触发时字段完整 | 事件触发时触发表的字段已写入（定理1） |
| 并发安全 | 同表多事件→标注 concurrent_trigger_risk |
