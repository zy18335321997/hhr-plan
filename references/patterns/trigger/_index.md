# 触发模式目录

> 从几建 338 个工作流蒸馏。覆盖四种触发方式的选择逻辑和组合模式。

## 自动选择表

| 场景 | 模式 | 触发方式 | 几建占比 |
|------|------|---------|---------|
| 人工决策/主动操作 | `button-trigger` | 自定义动作(按钮) | 37% |
| 数据变更自动联动 | `event-trigger` | 工作表事件 | 20% |
| 审批链下一个环节 | `approval-trigger` | 审批流程 | 19% |
| 可复用逻辑封装 | `subprocess-trigger` | 子流程调用 | 18% |
| 定期批量处理 | `schedule-trigger` | 定时触发 | 4% |
| 复杂编排 | `chained-trigger` | 多触发组合 | 2% |

## 触发方式决策树

```
操作涉及人工判断？
├─ 是 → button-trigger（公理2: 人在回路）
│       └─ 多人同操作？→ 每人独立按钮 or 共享按钮 + 角色权限
└─ 否 → 纯数据搬运？
        ├─ 数据变更后自动响应 → event-trigger
        │       └─ ⚠ 同一表多个事件触发？→ 检查并发冲突
        ├─ 审批后的下一步 → approval-trigger
        ├─ 定时批量 → schedule-trigger
        │       └─ ⚠ 频率≥每小时 → 考虑改为 event-trigger
        └─ 可复用逻辑 → subprocess-trigger
                └─ 深度≤2层
```

## 公理约束

| 约束 | 规则 |
|------|------|
| 公理2 | 涉及判断/决策 → 必须按钮触发，禁止定时/事件替代 |
| 公理5 | 事件触发 → 默认继续执行，不中止 |
| 定理1 | 事件触发的 T0 时刻，触发表的所有字段已存在（记录级触发） |
| 平台 | 同一表多事件触发 → 标注 concurrent_trigger_risk |
