# Subprocess-Trigger: 子流程调用

> 封装可复用逻辑为子流程，由父流程调用。减少重复配置。
> 几建占比: 18%（60个子流程调用）

## 场景

- 多个工作流共享同一段逻辑（如通知模板、日志写入、状态同步）
- 复杂工作流需要拆分（节点数>10 → VComplex → 子流程）
- 可独立测试的业务单元（审批链、计算逻辑、外部API调用）

## 配置模板

```
Tn: 子流程节点
    子流程名: [动词+名词]
    执行方式: 逐条执行 / 执行单次
    传入参数: [参数名→来源节点.字段]
    容错策略: 中止时继续下一条（公理5）
```

## 子流程决策

| 信号 | 决策 |
|------|------|
| 同一逻辑在≥3个工作流重复 | 封装为子流程 |
| 工作流节点数>10 | 拆分子流程（元公理A） |
| 逻辑独立、可单独测试 | 封装（公理1：封装复用） |
| 子流程嵌套深度>2 | 不嵌套，改为平行调用 |

## 子流程参数规范（公理3）

```
输入参数: 拼音缩写+类型（如 cgxqId / htmc / sqrName）
          └─ 禁止 param1/recordId 等无意义命名
输出: 子流程写回父流程关联记录
```

## 与 PBC (封装业务流程) 的区别

| | 子流程 | PBC |
|--|------|-----|
| 调用方式 | 父流程内节点 | 跨应用/跨表调用 |
| 参数传递 | 节点级 | API级 |
| 几建占比 | 18% | 2% |
| 复杂度 | Standard-Complex | VComplex |

## 公理约束

| 约束 | 规则 |
|------|------|
| 执行方式 | 逐条执行（多条时），非并行 |
| 失败策略 | 中止时继续下一条（公理5） |
| 嵌套深度 | ≤2层 |
| 参数命名 | 携带语义（公理3） |
| 批量上限 | 获取多条→子流程：最多 10,000 行 |
