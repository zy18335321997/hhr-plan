# Pattern D: 纯信号（Pure Signal）

> 不读不写，只做通知、提醒或状态推送。
> 数据流签名: 0 read → 0 write | 几建占比: 23.0%

---

## 场景

- 审批提醒（到期未处理→发通知）
- 状态变更通知（记录更新后→通知相关人）
- 定时汇总提醒（每周→发送本周待办汇总）
- 任何"不需要查数据、不需要改数据"的纯推送

## 最小节点链

```
T0: 触发([事件/定时]) → T1: 通知([目标人])
```

节点数: 2（Simple）。

## 常见变体

| 变体 | 节点链 | 节点数 | 分级 |
|------|-------|-------|------|
| 即时通知 | 事件触发→通知 | 2 | Simple |
| 延时提醒 | 事件触发→延时→通知 | 3 | Simple |
| 条件通知 | 事件触发→分支→通知A/通知B | 3-4 | Simple/Standard |

## 公理约束

| 节点 | 约束 |
|------|------|
| 通知 | 内容用【字段名】内联引用流程字段值（公理3）；不用默认消息模板 |
| 触发 | 纯推送场景用工作表事件或定时触发，不用按钮（公理2: 按钮留给人工决策） |

## 使用条件

- 无数据读写需求
- 纯通知/提醒/状态推送
- 目标节点数 ≤3

## 注意

- 界面推送(typeId=17): 一个工作流中仅第一个推送生效，后续被忽略
- 需要多次推送不同内容 → 拆成多个工作流，各自由事件触发
