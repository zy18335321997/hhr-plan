# 审批流程模式目录

> 从几建 65 个审批流蒸馏。覆盖审批链设计、审批人策略、退回/转审/加签模式。

## 自动选择表

| 场景 | 模式 | 关键特征 |
|------|------|---------|
| 单人审批，无层级 | `single-sign` | 1级或签，审批人固定或从字段取 |
| 多级审批（逐级） | `multi-level` | 2-3级或签，每级审批人可不同 |
| 条件审批（金额/类型分支） | `conditional-approval` | 小额自动通过，大额走审批 |
| 审批+抄送（知会） | `approval-with-cc` | 审批通过后自动通知相关人 |
| 审批退回重填 | `retreat-refill` | 退回仅发起节点，发起人修改后重提 |
| 审批超时自动处理 | `timeout-auto` | 限时处理+邮件通知+超时代理 |

## 公理约束总览

| 模式 | 公理2（人在回路） | 公理5（优雅降级） |
|------|-----------------|-----------------|
| single-sign | 按钮触发 | 审批人为空→拥有者代理 |
| multi-level | 每级或签 | 任一级超时→代理 |
| conditional-approval | 条件由人设定的字段决定 | 条件字段为空→默认走审批 |
| approval-with-cc | 通知仅知会，不参与决策 | 抄送失败不影响审批 |
| retreat-refill | 退回目标=仅发起节点 | 退回后工作流等待重提 |
| timeout-auto | 限时+邮件通知=开启 | 超时→拥有者代理，不卡住 |

## 几建证据

- 65 个审批流中，100% 使用或签（0 个会签）
- 审批人为空策略：100% 流程拥有者代理
- 退回目标：100% 仅发起节点
- 限时处理+邮件通知：默认开启
