## 校验模式速查

| 模式 | 节点链 | 空策略 | 失败行为 |
|------|-------|--------|---------|
| required-field-check | 触发→分支(字段为空?)→路径A(空:通知)/路径B(继续) | 字段为空→通知后继续 | 不中止，允许用户补填 |
| dedup-check | 触发→查询(同条件)→分支(有结果?)→有:通知+结束/无:新增 | 查询空→继续新增 | 发现重复→通知，不覆盖 |
| ref-existence-check | 触发→获取单条(精确查找)→分支(获取到?)→是:继续/否:通知 | 获取空→继续执行 | 关联不存在→通知，记录标记 |
| range-validation | 触发→分支(数值>上限?数值<下限?)→越界:通知/正常:继续 | 数值为空→默认值(公理5) | 越界→通知+标记 |
| state-precondition | 触发→获取(当前状态)→分支(状态匹配?)→是:操作/否:通知 | 状态字段为空→继续(默认初态) | 状态不匹配→通知，不操作 |
| parent-child-consistency | 查询主表→分支(主表存在?字段已填?)→是:子表操作/否:通知 | 主表缺失→通知+标记 | 先主后子(公理1) |

## 公理约束

| 约束 | 规则 |
|------|------|
| 公理1 | 每条记录可追溯→校验失败也写日志/标记，不静默跳过 |
| 公理5 | 校验失败→通知后继续执行（不中止流程） |
| 定理4 | 获取/查询的空数据处理策略=继续执行（不是中止） |
