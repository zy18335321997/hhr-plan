# 制造业 APaaS 设计模式

> 来源：莱尔德(LEDERP) 汽车零部件制造项目反向蒸馏 — 138表 / 427工作流 / 4,237字段
> 适用范围：制造型企业明道云项目（注塑/冲压/装配/汽配/电子制造等）

---

## 一、标准业务域模板

遇到制造类项目，默认按以下6域建表，再根据客户实际增减：

### 销售域（OTD: Order-to-Delivery）
```
客户 → 销售合同(+变更) → 销售订单(+明细) → 发货通知(+明细) → 销售退货(+明细) → 销售账单
```
- 可选：销售预测、销售价目清单、客户联系人、收货地址

### 采购域（P2P: Procure-to-Pay）
```
供应商 → 采购合同(+变更) → 采购预测 → 采购计划(+明细) → 采购订单(+明细) → 收货单(+明细) → 采购退货(+明细) → 采购账单
```
- 可选：采购价目清单、需求池、到货计划

### 生产域（MES核心）
```
生产预测 → 生产计划(+商品+物料清单) → 生产工单(+物料明细) → 生产领料(+明细) → 生产报工 → 不良记录 → 生产退料
```
- 可选：生产数据采集、工单产能记录、齐套计算、批量派工/报工

### 仓储域（WMS核心）
```
仓库 → 库位 → 批次 → 总库存 → 仓库库存 → 库位库存 → 批次库存 → 库存流水 → 库存清单
```
- 可选：收货批次清单、发货批次清单、移库单(+明细)、盘点单(+明细)、调拨单(+明细)、库存初始化

### 技术域（PLM核心）
```
物料 → BOM(+变更) → 工序 → 工艺参数卡(+明细) → 图纸管理
```
- 可选：班组、车间、员工、数据采集组

### 设备域（EAM核心）
```
机台 → 设备管理 → 设备保养计划 → 保养单(+项目明细) → 巡检单(+部位明细) → 报修单
```
- 可选：设备实时快照、设备时段汇总、物联设备、设备告警

## 二、标准字段字典

### 2.1 每张业务表必备字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 备注 | Text | 手工备注，每张业务表都需要 |
| 附件 | Attachment | 支持上传相关文件 |

> 系统字段（明道云内置）：创建人、创建时间、拥有者、最近修改人、最近修改时间。这些不需要手动建，但设计时要考虑。

### 2.2 高频业务字段（跨表复用规范）

| 字段名 | 类型 | 适用场景 | 出现频次 |
|--------|------|---------|---------|
| 审批状态 | Dropdown | 需要审批的单据 | 29张表 |
| 结单状态 | Dropdown | 业务单据关闭标记 | 26张表 |
| 物料 | Relation→物料 | 涉及物料的明细/清单 | 42张表 |
| 物料名称 | Text | 物料只读展示 | 49张表 |
| 物料大类 | Dropdown | 物料分类 | 40张表 |
| 规格型号/图号 | Text | 物料规格 | 28张表 |
| 客户 | Relation→客户 | 销售相关表 | 25张表 |
| 供应商 | Relation→供应商 | 采购相关表 | 20张表 |
| 制单日期 | Date | 单据创建日期 | 23张表 |
| 制单人 | Collaborator | 单据创建人 | 21张表 |
| 生产工单 | Relation→生产工单 | 生产相关子表 | 23张表 |
| 单位 | Dropdown | 物料计量单位 | 53张表 |

### 2.3 命名规则

- 关联字段名 = 目标表名（如关联到"物料"表，字段就叫"物料"，不要叫"物料ID"或"关联物料"）
- 布尔字段 = `是否XX`（如"是否启用""是否完成"）
- 日期字段 = `XX日期`（如"制单日期""交货日期"）
- 状态字段 = `XX状态`（如"审批状态""结单状态""使用状态"）
- 金额字段 = `XX金额`（如"预算金额""含税金额"）

## 三、表单分层架构

### Hub主表（含SubTable）
- 特征：AutoNumber单号 + SubTable子表 + Dropdown状态 + Collaborator人员
- 示例：销售订单、采购订单、生产工单、发货单、收货单
- 占比：约28%的表

### 子表明细表（被SubTable引用）
- 特征：Relation字段占比高（指向物料/主表等），字段数适中
- 示例：销售明细、采购明细、发货明细、领料明细
- 命名规律：`{主表名}明细` 或 `{业务}清单`

### 配置/基础数据表
- 特征：字段数少(≤15)，以Dropdown+Text为主
- 示例：仓库、库位、班组、车间、不良分类、不良原因

### 计算汇总表
- 特征：含Rollup/Formula/Lookup字段
- 示例：总库存、仓库库存、库位库存、齐套计算结果
- 命名规律：`{维度}库存`、`{维度}汇总`

## 四、工作流设计模式（制造专用）

### 4.1 运算密集型工作流
- 莱尔德 typeId=2(运算节点) : CRUD节点 = 1821:1344 ≈ 1.35:1
- 大量使用赋值/计算/条件判断做数值运算
- 适用场景：库存计算、齐套算料、BOM展开、成本核算

### 4.2 子流程复用
- 子流程占比：121/427 = 28%
- 拆分层级：父流程(306条) + 子流程(121条)，≤2层嵌套
- 适用模式：算料逻辑、BOM展开、库存更新等通用逻辑包装成子流程

### 4.3 异常处理
- 每流1.7个异常处理节点(720个/427条)
- 异常处理节点(typeId=13)放在每个CRUD/查询节点之后
- 确保单点失败不中断整个流程链

### 4.4 工作流命名
- 莱尔德风格：`【工作表名】动作描述`（如`【库位清单】出/入库`）
- 或用简洁动词：`拉取BOM清单`、`执行算料`、`刷新账单`

## 五、设计原则速查

| # | 原则 | 来源 | 做法 |
|---|------|------|------|
| 1 | 关联驱动 | 91%表含Relation | 先画ER图，再建表 |
| 2 | 审计追踪 | DateTime(859)+Collaborator(755) | 业务表必备创建人/时间+状态 |
| 3 | 主子不分家 | 38张Hub表含SubTable | 单据=主表(单号+状态)+子表(明细) |
| 4 | 域隔离 | 9大域物理分区 | 域内闭环，域间Hub桥接 |
| 5 | 字段名即文档 | 同概念同名称 | 关联字段名=目标表名 |
| 6 | 编号体系 | 53张表含AutoNumber | 前缀+日期+流水 |
| 7 | 运算前置 | typeId=2 > CRUD | 先算再写，不要边写边算 |
| 8 | 异常必处理 | 每流1.7个异常处理 | CRUD后必跟异常处理 |

## 六、快速启动清单

遇到新的制造类项目时，按此清单顺序搭建：

- [ ] 1. 确认业务域数量（标准6域：销售/采购/生产/仓储/技术/设备）
- [ ] 2. 每域建Hub表（含AutoNumber + Dropdown状态 + Collaborator）
- [ ] 3. Hub表下建子表（命名=主表+明细，含Relation→物料/主表）
- [ ] 4. 建基础数据表（仓库/库位/班组/车间/不良分类等）
- [ ] 5. 建计算汇总表（库存=Rollup入库-出库）
- [ ] 6. 交叉关联（子表→主表，主表→客户/供应商）
- [ ] 7. 工作流：先建子流程（算料/库存更新），再建主流程（CRUD+通知）
- [ ] 8. 每流加异常处理节点
- [ ] 9. 补齐审批链路（审批状态+审批块+通知）
- [ ] 10. 填充测试数据，验证闭环
