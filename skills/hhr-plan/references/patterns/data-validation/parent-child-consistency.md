# Parent-Child Consistency: 主子表一致性

> 确保子表操作（新增/修改/删除）的主表引用始终有效。
> 节点签名: 先主后子，子回写父引用。公理1的核心实现。

## 场景

- 创建子记录前确认主记录存在
- 父记录状态变更→子记录状态同步
- 子记录删除→父记录汇总更新
- 主子表的级联操作

## 创建子记录

```
T0: 按钮触发(主表) → T1: 获取主表记录(当前记录)
→ T2: 新增子表记录
    字段映射: 父引用←主表.记录ID
            状态←默认值(公理5)
            创建人←触发人
```

## 批量子记录操作

```
T0: 事件触发(子表变更) → T1: 获取父记录(通过子表.父引用)
→ T2: 分支(父记录存在?)
    ├─ 存在 → T3a: 获取多条子记录 → T4a: 汇总计算 → T5a: 更新主表
    └─ 不存在 → T3b: 通知"父记录不存在"（异常）
```

## 硬规则

1. **先主后子**：创建子记录前必须确认主记录存在
2. **子回写父**：子记录必须保存父引用（记录ID）
3. **汇总更新**：子记录批量变更后重新计算主表汇总字段
4. **级联保护**：主记录删除前检查是否有子记录

## 公理约束

| 约束 | 规则 |
|------|------|
| 创建顺序 | 先建主记录，后建子记录（公理1） |
| 父引用 | 子记录必须回写父引用记录ID（公理1） |
| 汇总方式 | 直接计算（缓存），非每次动态获取（公理1） |
| 空值处理 | 查询父记录空→继续执行+通知（公理5） |
| 子表上限 | 逐条执行，中止继续下一条（公理5） |
