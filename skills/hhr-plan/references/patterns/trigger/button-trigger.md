# Button-Trigger: 按钮触发

> 人工决策→自定义动作(按钮)。公理2的核心载体。
> 几建占比: 37%（126个按钮触发工作流）

## 场景

- 审批类工作流
- 主动创建的记录（新增采购需求、发起合同）
- 手动触发的操作（发布任务、发送通知、更新状态）
- 任何需要"人判断后执行"的操作

## 配置模板

```
T0: 按钮触发
    工作表: [触发表]
    按钮名: [动词+名词，如"提交审批"/"发布任务"/"确认收货"]
    数据源: 单条（从当前记录触发）
```

## 按钮命名规范（公理3）

| 动作 | 命名 | 示例 |
|------|------|------|
| 提交审批 | 提交审批 | 提交请假审批 |
| 发布/下发 | 发布XX | 发布任务、发布采购需求 |
| 确认操作 | 确认XX | 确认收货、确认验收 |
| 撤销/取消 | 撤销XX | 撤销申请、取消订单 |
| 发起流程 | 发起XX | 发起合同、发起询价 |

> 按钮名=动词+名词，不可只用"提交"或"操作"。

## 与事件触发的区别

| | 按钮触发 | 事件触发 |
|--|---------|---------|
| 触发时机 | 人主动点击 | 数据变更自动 |
| 适用 | 涉及判断/决策 | 纯数据级联 |
| 公理 | 公理2: 人在回路 | 公理5: 自动响应 |
| 几建占比 | 37% | 20% |

## 公理约束

| 约束 | 规则 |
|------|------|
| 涉及决策→必须按钮 | 不允许用定时/事件替代（公理2） |
| 按钮可见性 | 按角色/状态配置可见条件 |
| 操作日志 | 按钮触发自动记录操作人+时间 |
