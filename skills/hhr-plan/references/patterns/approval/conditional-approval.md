# Conditional-Approval: 条件审批

> 根据金额/类型/优先级等字段条件，走不同审批路径。小额自动通过，大额多级审批。
> 节点签名: 触发→分支(条件)→路径A(自动通过)/路径B(审批链) | 节点数: 4-7

## 场景

- 报销金额<500→自动通过，500-5000→部门审批，>5000→部门+财务
- 采购类型=办公用品→自动通过，=设备→多级审批
- 紧急程度=普通→标准审批，=紧急→快速通道(1级审批)

## 最小节点链

```
T0: 按钮触发 → T1: 分支(金额≤阈值?)
├─ 是 → T2a: 记录"自动通过"标签 → T3a: 通知
└─ 否 → T2b: 获取审批人 → T3b: 审批 → T4b: 通知
```

节点数: 4-7 (Standard)。

## 变体

| 条件类型 | 分支字段示例 | 分级 |
|------|------------|------|
| 金额条件 | 报销金额/合同金额/采购金额 | Standard |
| 类型条件 | 采购类型/项目类型/合同类型 | Standard |
| 优先级条件 | 紧急程度/处理优先级 | Standard |
| 多条件组合 | 金额+类型（分支嵌套） | Complex（建议拆分） |

## 公理约束

| 约束 | 规则 |
|------|------|
| 分支条件 | 覆盖目标字段所有可能值（逻辑完备性） |
| 条件字段 | 必须设默认值（公理5） |
| 自动通过 | 仍需写日志/标签，保证数据血缘（公理1） |
| 审批路径 | 同 single-sign 或 multi-level 的约束 |
| 分支嵌套 | >1层嵌套 → 拆分为独立工作流 |
