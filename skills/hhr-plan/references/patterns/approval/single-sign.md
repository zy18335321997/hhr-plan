# Single-Sign: 单人审批

> 1级或签，审批人固定或从字段取。最常见的审批模式。
> 节点签名: 触发→获取→审批→通知 | 几建占比: ~40%（审批流中）

## 场景

- 请假/报销/采购申请→直属上级审批
- 审批人从"直属上级"字段取，或固定为流程拥有者
- 审批结果通知发起人

## 最小节点链

```
T0: 按钮触发 → T1: 获取审批人(从字段/固定) → T2: 审批(或签) → T3: 通知
```

节点数: 3-5 (Simple/Standard)。

## 变体

| 变体 | 节点链 | 节点数 |
|------|-------|-------|
| 审批人固定 | 触发→审批→通知 | 3 |
| 审批人从字段 | 触发→获取→审批→通知 | 4 |
| 审批+更新状态 | 触发→获取→审批→更新→通知 | 5 |

## 公理约束

| 节点 | 约束 |
|------|------|
| 触发 | 按钮触发（公理2：人工决策→按钮） |
| 审批模式 | 或签（1人通过即可） |
| 审批人为空 | 由流程拥有者代理（公理5） |
| 限时+邮件 | 必须开启（公理3） |
| 退回目标 | 仅发起节点（公理2） |
| 通知 | 自定义模板（公理3），【字段名】内联引用 |

## 几建证据

- 所有审批流使用或签
- 审批人为空→100% 流程拥有者代理
- 限时处理+邮件通知默认开启
