# Multi-Level: 多级审批

> 2-3级逐级审批，每级或签。上级审批通过后自动进入下一级。
> 节点签名: 触发→审批(L1)→审批(L2)→[审批(L3)]→通知 | 节点数: 5-8

## 场景

- 采购金额>10万→需要部门负责人→财务→总经理三级审批
- 合同审批→法务→商务负责人→CEO
- 每一级审批人在上一级通过后自动获取

## 最小节点链

```
T0: 按钮触发 → T1: 获取L1审批人 → T2: 审批L1(或签)
→ T3: 分支(L1通过?) → T4: 获取L2审批人 → T5: 审批L2
→ T6: 通知(通过/否决)
```

节点数: 5-8 (Standard/Complex)。

## 变体

| 变体 | 节点链 | 节点数 | 分级 |
|------|-------|-------|------|
| 固定二级 | 触发→审批L1→审批L2→通知 | 5-6 | Standard |
| 条件三级 | 触发→审批L1→分支(金额>X)→审批L2→[审批L3]→通知 | 6-8 | Complex |
| 会签变体 | 不推荐。或签是公理2约束。如必须会签→标注违反公理2。 |

## 公理约束

| 约束 | 规则 |
|------|------|
| 触发 | 按钮触发（公理2） |
| 每级模式 | 或签，非会签（公理2） |
| 每级限时 | 必须配置限时+邮件通知（公理3） |
| L1通过后 | 自动获取L2审批人，不等人手动指定 |
| 任一级否决 | 直接通知发起人+结束流程 |
| 审批链完整 | L1→L2→L3，不允许跳级 |
| 子流程 | 节点数>8 → 每级拆分为独立子流程 |
