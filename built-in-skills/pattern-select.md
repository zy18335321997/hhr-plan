# 模式库匹配

> 从需求自动匹配已有的可复用业务模块模板。匹配到 → 标注覆盖率和需定制部分；未匹配 → 提示可能的新模式候选。

## 匹配流程

### Step 1: 实体识别

从用户需求中提取核心实体：
- 合同/项目/客户/任务/费用/人员/物品/车辆/...

### Step 2: 模式库检索

查 `references/patterns/modules/` 目录：
- 模块名匹配
- 实体名匹配
- 关键词匹配（如"审批"→匹配审批链模板、"通知"→匹配通知模板）

### Step 3: 覆盖率估算

加载匹配到的 pattern 文件，逐项对比用户需求：
- 已覆盖：pattern 中有、需求也有的部分
- 需定制：需求有但 pattern 中无对应结构的部分
- 不需：pattern 中有但需求不涉及的部分

### Step 4: 输出

```
匹配结果: [模块名]（XX% 覆盖）
  可直接复用:
    - [表/流/字段名]
  需定制:
    - [需求中但模块模板不包含的部分]
  参考建议:
    - [几建中的对应部分或类似实现]
  公理对齐:
    - [该模块涉及的公理约束]
```

如果多条匹配 → 列表展示，让用户选择。
如果无匹配 → 提示"此为新建模式，建议设计完成后沉淀为模板"。

## 沉淀触发

当 Mode A/B 完成后，如果设计方案覆盖了 `references/patterns/modules/` 中不存在的业务场景，询问用户是否沉淀为新模板。

## 交互

模式库匹配不是阻断步骤——匹配到就给参考，没匹配到就跳过。不因为没模板而中断设计流程。
